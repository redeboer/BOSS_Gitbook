Detailed Description

Base algorithm that creates a selection of charged and/or neutral tracks.

Your analysis has to be defined in derived algorithms, making use of the vectors that are filled in this algorithm. See the tutorial pages for more information.

Todo:
The design of using maps of NTuple::Itemss is not ideal, as it does not (yet) allow enough flexibility.

void TrackSelector::BookNTuple ( NTupleContainer &  tuple)
private

Helper function that allows you to relate the NTupleContainer argument tuple to the output file (i.e.

to 'book' it).

  1. Form a string for booking in the file. Typically: "FILE1/<tree name>".

  2. Attempt to get this NTuple::Tuple from file the file.

  3. If not available in file, book a new one.

  4. Add the booked NTuple::Tuple to the NTuple mapping (NTupleContainer::instances). Note: Use map::operator[] if you want to book an NTuple::Item and use map::at if you want to access the NTuple by key value. This ensures that the programme throws an exception if you ask for the wrong key later.

This function encapsulates the addItem procedure for the energy loss branch ("dedx").

This method allows you to perform the same booking method for different types of charged particles (for instance 'all charged particles', kaons, and pions).

  1. "Ngoodhits": Number of good hits (excluding overflow).

  2. "Ntotalhits": Number of good hits (including overflow).

  3. "chie": in case of electron ("number of \f$\sigma\f$ from \f$e^\pm\f$").

  4. "chik": in case of kaon ("number of \f$\sigma\f$ from \f$K^\pm\f$").

  5. "chimu": in case of muon ("number of \f$\sigma\f$ from \f$\mu^\pm\f$").

  6. "chip": in case of proton ("number of \f$\sigma\f$ from \f$p^\pm\f$").

  7. "chipi": in case of pion ("number of \f$\sigma\f$ from \f$\pi^\pm\f$").

  8. "normPH": Normalized pulse height.

  9. "p": Momentum of the track as reconstructed by MDC.

  10. "probPH": Most probable pulse height from truncated mean.

◆ execute()

StatusCode TrackSelector::execute ( )

Inherited execute method of the Algorithm which is called for each event.

  1. Load next event from DST file. For more info see:

    • Namespace EventModel
    • Class EvtRecEvent
    • Type definition EvtRecTrackCol

    • Type definition Event::McParticleCol

  2. Get Monte Carlo truth if available (that is, if the run number is negative).

    1. Load McParticelCol from "/Event/MC/McParticleCol" directory in FILE1.

    2. Abort if does not exist.

      Todo:
      Might have to make this aborting behaviour less strict.

    3. Loop over collection of MC particles (Event::McParticleCol). For more info on the data available in McParticle, see here. Only add to fMcParticles if the McParticle satisfies:

      • It is not a primary particle.

      • Only add if the decay has been generated.

        Todo:
        Why? What does this mean precisely?

      • Only start recording after we have passed the initial cluster (e.g. ). Note that 91 is the PDG code of cluster and 92 is the PDG code of string.

      • 91 is the PDG code of cluster.

      • 92 is the PDG code of string.

      • Skip if this particle is the initial cluster itself.

      • Add the pointer to the fMcParticles vector.

  3. Create selection charged tracks and write track info:

    Note: the first part of the set of reconstructed tracks are the charged tracks.

    1. Get MDC information

      Todo:
      Check if helixip.pivot(fVertexPoint) doesn't affect fVertexPoint.

    2. Apply charged track cuts:

      • coordinate of the collision point has to be within cut_Vz0_max

      • radius in plane has to be less than cut_Vr0_max

      • coordinate of the distance to the interaction point has to be within cut_Rvz0_max

      • distance to the interaction point in plane has to be within cut_Rvxy0_max

      • distance to the interaction point in plane has to be within cut_costheta_max

    3. Write charged track vertex position info ("charged" branch)

    4. Write dE/dx PID information ("dedx" branch)

    5. Write Time-of-Flight PID information ("tof*" branch)

  4. Create selection of neutral tracks and write track info.

    Note: The second part of the set of reconstructed events consists of the neutral tracks, that is, the photons detected by the EMC (by clustering EMC crystal energies). Each neutral track is paired with each charged track and if their angle is smaller than a certain value (here, 200), the photon track is stored as 'good photon' (added to iGam).

    1. Get EMC information

    2. Apply photon energy cut (set by TrackSelector.cut_PhotonEnergy).

    3. Write neutral track information (if write_neutral is set to true).

    4. Add photon track to vector of neutral tracks (fGoodNeutralTracks).

  5. Write event info ("mult" branch)

  6. Write event info ("vertex" branch)

  7. Perform derived algoritm as defined in TrackSelector::execute_rest.

◆ initialize()

StatusCode TrackSelector::initialize ( )

(Inherited) initialize step of Algorithm.

This function is called once in the beginning of each run. Define and load NTuples here. The NTuples will become the TTrees in the eventual ROOT file, the added NTuple::Items will be the branches of those trees.

  1. "mult": Multiplicities of the total event

    1. "Ntotal": Total number of events per track.

    2. "Ncharge": Number of charged tracks.

    3. "Nneutral": Number of charged tracks.

    4. "NgoodNeutral": Number of 'good' neutral tracks.

    5. "NgoodCharged": Number of 'good' charged tracks.

    6. "Nmdc": Number of charged tracks in MDC.

  2. "vertex": Vertex info

    1. "vx0": coordinate of the collision point.

    2. "vy0": coordinate of the collision point.

    3. "vz0": coordinate of the collision point.

  3. "charged": Charged track info.

    1. "vx": coordinate of the secondary vertex as determined by MDC.

    2. "vy": coordinate of the secondary vertex as determined by MDC.

    3. "vz": coordinate of the secondary vertex as determined by MDC.

    4. "vr": Distance from origin in plane.

    5. "rvxy": Nearest distance to interaction point in plane.

    6. "rvz": Nearest distance to interaction point in direction.

    7. "rvphi": Angle in the plane.

      Todo:
      Get explanation of geometry (angle) definitions in an event.

    8. "phi": Helix angle of the particle.

    9. "p": Momentum of the track.

  4. "neutral": Neutral track info.

    1. "E": Energy of the neutral track as determined by the EM calorimeter.

    2. "x": -coordinate of the neutral track according to the EMC.

    3. "y": -coordinate of the neutral track according to the EMC.

    4. "z": -coordinate of the neutral track according to the EMC.

    5. "phi": -angle of the neutral track according to the EMC.

    6. "theta": -angle of the neutral track according to the EMC.

    7. "time": Time of the neutral track according to the EMC.

      Todo:
      Investigate what this parameter precisely means.

  5. "dedx": energy loss PID branch. See TrackSelector::BookNtupleItems_Dedx for more info.

  6. "ToFEC", "ToFIB", and "ToFOB": information from the three Time-of-Flight detectors. See TrackSelector::BookNtupleItems_Tof for more info.

  7. "PID": Track PID information.

    1. "p": Momentum of the track as reconstructed by MDC.

    2. "cost": Theta angle of the track.

    3. "chiToFIB": of the inner barrel ToF of the track.

    4. "chiToFEC": of the end cap ToF of the track.

    5. "chiToFOB": of the outer barrel ToF of the track.

    6. "chidEdx": of the energy loss of the identified track.

    7. "prob_K": Probability that the track is from a kaon according to the probability method.

    8. "prob_e": Probability that the track is from a electron according to the probability method.

    9. "prob_mu": Probability that the track is from a muon according to the probability method.

    10. "prob_p": Probability that the track is from a proton according to the probability method.

    11. "prob_pi": Probability that the track is from a pion according to the probability method.
