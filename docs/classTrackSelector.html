Parameters
tupleNameThis will not only be the name of your NTuple, but also the name of the TTree in the output ROOT file when you use NTuple::write(). The name used here is also used as identifier in NTuplePtr map fNTupleMap. In other words, you can get any of the NTuplePtrs in this map by using fNTupleMap[<tuple name>]. Inherited execute method of the Algorithm.

This function is called for each event.

STEP (A): Load next event from DST file

STEP (B): Create selection charged tracks and write track info

STEP 1: Get MDC inform

STEP 2: Apply vertex cuts, store

STEP 3: WRITE primary vertex position info ("vxyz" br

STEP 4: WRITE dE/dx PID information ("dedx" branch)

STEP 5: WRITE Time-of-Flight PID information ("tof*" branch)

STEP (C): WRITE event info ("mult" and "vertex" branch)

STEP (D): Perform derived algoritm

NTuples are like vectors, but its members do not necessarily have to be of the same type.

In this package, the NTuples are used to store event-by-event information. Its values are then written to the output ROOT file, creating a ROOT TTree. In that sense, each NTuple here represents one TTree within that output ROOT file, and each NTuple::Item represents its leaves. The name of the leaves is determined when calling NTuple::addItem. Note that the NTuple::Items have to be added to the NTuple during the TrackSelector::initialize() step, otherwise they cannot be used as values! This is also the place where you name these variables, so make sure that the structure here is reflected there! Package property that determines whether or not to record multiplicities.
