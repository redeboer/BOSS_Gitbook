Detailed Description

Helper class for the topoana package.

It is essential to run load this algorithm along with your own algorithm if you want to run topoana.exe over the output of your initial event selection. This algorithm (MctruthForTopoAna) will store the relevant 'Monte Carlo truth' information.

Remarks
January 21st, 2019: This procedure has been implemented in the TrackSelector and does not have to be loaded anymore.

StatusCode MctruthForTopoAna::execute ( )

  1. Create log stream object and increase counter for number of events.

  2. Load event from DST file and get run and event number.

  3. Attempt to get 'Monte Carlo truth'.

  4. Attempt to get 'Monte Carlo truth'.

    • Reset the m_nmcps counter for the number of MC particles.

    • 91 is the PDG code of cluster.

    • 92 is the PDG code of string.

  5. Loop over collection of MC particle objects. An array containing a selection of MC truth particles is created here.

    • Only decayed particles are of interest to the topoana package.

    • The rootIndex variable is used to skip particles like the boson in the decay chain after the collision. It has to be stored once and untill it is stored, all MC truth particles are rejected (which is what the incPdcy boolean is for).

    • Skip if this particle is the initial cluster itself.

    • Store PDG code of this MC particle.

    • Store track ID of the mother. This ID is -1 if it is the initial cluster (e.g. ).

  6. Write info stored in the loop to the "MctruthForTopoAna" TTree.

◆ initialize()

StatusCode MctruthForTopoAna::initialize ( )

Book NTuple for MC truth.

The contents of this NTuple are required for the topoana package to function properly.

  1. Create log stream (MsgStream object).

  2. Book NTuple called "MctruthForTopoAna". A selection of MC truth data will be written to this TTree. This tree contains:

    • "Ievt": counter for number of events (not the ID!).

    • "Runid": run number ID.

    • "Evtid": event number ID.

    • "Nmcps": number of MC particles stored for this event. This one is necessary for loading following two items, because they are arrays.

    • "Pid" (array): PDG code for the particle in this array.

    • "Midx" (array): track index of the mother particle (corrected with rootIndex).

  3. Abort if booking the NTuple has failed.

  4. Reset counter for number of events.
